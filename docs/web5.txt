
[ web5 ]

1. 프로젝트 개요
    목적:  Spring Boot와 Spring Data JPA 기반의 회원제 게시판(Board) 서비스를 직접 구축하며 MVC 패턴의 핵심 CRUD를 복습.
        더불어 실무 필수 요소인 파일 입출력(I/O), Spring Security 인증/인가 처리,
        JPA 연관 관계 및 고도화된 페이징/검색 기능을 체계적으로 마스터하는 것을 목표로 한다.

    개발 환경 및 구조:
        Backend: Java 21, Spring Boot 4.x (Spring Framework 7.x 기반), Spring Data JPA, Spring Security 6.x
        Database: PostgreSQL (자동 증가 Identity 전략, 가비지 데이터 방지를 위한 Cascading 및 외래키 제약조건 설정)
        Frontend: Thymeleaf HTML5, CSS3, JavaScript
        패키지 구조: net.datasa.web5 표준 레이어드 아키텍처 기반
            controller / service / repository / domain (dto, entity) / security / util


2. 주요 실습 내용
    ① 회원 관리 기능 및 Spring Security (인증과 인가)
        뷰 폴더:  templates/memberView/, templates/admin/
        실습 파일:
            MemberController, AdminController, MemberService,
            AuthenticatedUser, AuthenticatedUserDetailsService, WebSecurityConfig
            loginForm.html, joinForm.html, infoForm.html, inactive.html, adminPage.html, memberList.html
        내용:
            BCrypt 암호화:
                회원가입 시 BCryptPasswordEncoder를 활용해 비밀번호를
                단방향 해시 암호화하여 DB(member 테이블)에 안전하게 저장.
            시큐리티 로그인/로그아웃:
                폼 로그인을 통해 사용자가 입력한 아이디와 비밀번호를
                UserDetailsService 인터페이스 구현체에서 가로채 DB 데이터와 매칭 및 인증 과정을 수행.
                로그아웃 시 세션 무효화와 쿠키 파괴를 자동화.
            권한 기반 제한:
                계정 상태(enabled: 사용 가능 1, 사용 불가능 0)와 사용자 구분(rolename: ROLE_USER, ROLE_ADMIN)에 따른
                접근 제어를 완벽하게 구현합. 관리자 기능은 일반 사용자가 URL로 직접 쳐서 들어오지 못하도록 원천 차단.

    ② 첨부파일 및 이미지 처리를 포함한 게시판 CRUD
        뷰 폴더:  templates/boardView/, templates/
        실습 파일:
            BoardController, BoardService, BoardRepository, ReplyRepository, FileManager
            list.html, listAll.html, read.html, writeForm.html, updateForm.html, replyList.html, home.html
        내용:
            가변형 본문 처리: 수천 자 이상의 긴 글 생성을 위해 DB의 TEXT 타입과 엔티티를 매핑하여 온전한 게시글 CRUD를 수행.
            파일 업로드/다운로드: 서버 내부의 특정 안전 디렉토리에 파일을 저장하고,
                원본 파일명(original_name)과 중복을 피하기 위해 난수화하여 물리 저장되는 파일명(file_name)을 분리해 관리.
                FileManager 유틸리티와 Java I/O Stream을 이용해 파일 다운로드 및 실시간 이미지 미리보기(preview) 기능을 제공.


3. 핵심 동작 흐름 (Flow)
    ① Spring Security 인증 및 인가 처리 흐름
        사용자 요청: 사용자가 로그인이 필요한 메뉴(예: 글쓰기)를 요청하거나 로그인 폼을 제출.
        시큐리티 필터 작동: WebSecurityConfig에 정의된 SecurityFilterChain이 가로채 PUBLIC_URLS인지 먼저 확인함.
            인증이 안 된 상태라면 로그인 페이지(/member/loginForm)로 리다이렉트.
        인증 수행:
            로그인 제출 시 AuthenticatedUserDetailsService가 구동되어 DB에서 해당 아이디를 조회하고,
            AuthenticatedUser 객체(UserDetails 구현체)에 회원 정보 및 권한 리스트를 담아 시큐리티 컨텍스트에 저장.
        인가 판별 (View -> Config -> Controller 3단계 철통 보안):
            1단계 (View): Thymeleaf 시큐리티 문법을 통해 로그인 전에는 [로그인], 로그인 후에는 [로그아웃/마이페이지]가 보이게 하며,
                관리자(ROLE_ADMIN)에게만 [관리자 페이지] 버튼이 렌더링되도록 동적 UI를 제어.
            2단계 (Config): WebSecurityConfig 내 requestMatchers() 설정을 통해 특정 주소 패턴의 물리적 접근 권한을 제한.
            3단계 (Controller): @EnableMethodSecurity를 켜고,
                관리자 컨트롤러나 메서드 상단에 @PreAuthorize("hasRole('ROLE_ADMIN')") 어노테이션을 붙여
                API 진입 직전에 최종적으로 권한을 체크함. 권한이 없다면 403 Forbidden 에러 페이지로 가로챔.

    ② 첨부파일 업로드 및 이미지 미리보기 흐름
        클라이언트 요청: 글쓰기 폼(writeForm.html)에서 enctype="multipart/form-data" 속성을 지정하고 파일을 첨부하여 제출.
        컨트롤러 수신: MultipartFile 객체로 파일을 바인딩받아 FileManager 유틸 클래스로 전달.
        물리 저장 및 DB 기록: UUID 등을 결합해 유일한 물리 파일명을 생성하여 서버 지정 폴더에 저장하고,
            영속성 컨텍스트를 거쳐 board 테이블의 original_name과 file_name 컬럼에 정보를 세팅.
        이미지 미리보기 및 다운로드:
            브라우저가 이미지 태그(<img src="/board/preview?num=...">)를 만나면
            서버는 FileInputStream을 열어 디스크의 물리 파일을 바이트 배열(byte[])로 읽어 들인 뒤,
            Response Body의 OutputStream에 부어 브라우저가 이미지를 그대로 그리게 유도함.
            다운로드 역시 이와 유사하게 Response Header 설정을 조작해 파일 다운로드 창이 뜨도록 구현함.


4. 실습 포인트
    JPA 엔티티 기본키 생성 전략:
        MySQL의 AUTO_INCREMENT에 맞게 엔티티 기본키 속성에 @GeneratedValue(strategy = GenerationType.IDENTITY)를 지정.
        데이터 유실이나 정수형 번호 꼬임(시퀀스 싱크 불일치) 현상을 방지하는 정석적인 매핑법을 학습.

    JPA 연관 관계 설정과 전파 제어:
        Member(1) ↔ Board(N), Board(1) ↔ Reply(N) 관계를 @ManyToOne과 @OneToMany를 활용하여 양방향/단방향으로 매핑.
        특히 부모 글이 지워지면 자식 댓글도 함께 연쇄 삭제되도록 cascade = CascadeType.ALL과
        고아 객체를 완전 격리하는 orphanRemoval = true 속성의 메커니즘을 명확히 이해.

    JPA 무한 순환 참조(StackOverflow) 차단:
        양방향 연관 관계에서 객체를 단순히 출력하거나 JSON으로 변환할 때 Board가 Reply를 부르고,
        Reply가 다시 Board를 호출하면서 메모리가 터지는 순환 참조 문제를 예방하기 위해,
        Lombok 사용 시 연관 관계 필드를 제외하는 @ToString(exclude = "...") 설정을 명확히 적용.

    JPA 페이징, 정렬, 검색:
        컨트롤러에서 Pageable 객체를 받아 파라미터를 넘기고,
        Repository에서 Page<BoardEntity> 형태로 받아와 타임리프 뷰 단에
        하단 페이징 네비게이션바를 동적으로 그리는 실무형 페이징 로직을 학습.

    쿼리 고도화:
        JPA가 메서드 이름을 보고 쿼리를 자동 생성해 주는 "메서드 네이밍 규칙"을 학습하고,
        복잡한 조건 검색이나 다중 조인을 처리하기 위해 엔티티 객체 중심의
        SQL인 JPQL(Java Persistence Query Language)을 학습.


** 알아두기 **
📌 JPA 엔티티 설계 시 @Data 어노테이션 사용 금지!
    롬복의 편리한 기능인 @Data는 내부적으로 모든 필드를 포함하여 equals(), hashCode(), toString() 메서드를 자동으로 오버라이딩함.
    이 상태에서 연관 관계 매핑(@ManyToOne, @OneToMany)이 걸려 있으면,
    하이버네이트가 기동하거나 데이터를 조회할 때 상호 호출을 유발하여 시스템이 터지거나 영속성 유닛 바인딩 에러가 발생함.
    엔티티 객체에는 반드시 @Getter와 @Setter를 각각 명시하고,
    연관 관계 필드는 @ToString(exclude = "필드명")로 격리하는 것이 표준.

📌 DB 데이터 타입과 자바 래퍼(Wrapper) 클래스의 1:1 매핑 규칙 준수!
    스프링 부트 환경에서 ddl-auto=validate 옵션이 켜져 있으면 자바 코드의 필드 타입과
    실제 가동 중인 DB 테이블의 컬럼 속성을 엄격하게 대조함.
    예를 들어 PostgreSQL의 smallint 타입 컬럼을 자바의 Boolean으로 매핑하려고 하면
    하이버네이트 유효성 검증 단계에서 예외를 던지며 구동에 실패함.
    참/거짓 상태값을 나타낼 때는 DB 테이블도 boolean 타입, 자바 엔티티도 Boolean 타입으로 일치시켜 주거나,
    숫자로 관리할 때는 DB의 smallint와 자바의 Integer/Short를 매치시켜 타이핑 미스매치 오류를 원천 차단해야 함.
